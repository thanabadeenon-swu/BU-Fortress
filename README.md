## Problem  
ในโครงสร้างพื้นฐานไอทีสมัยใหม่ ปริมาณของข้อมูล Log ที่เกิดขึ้นจากระบบเครือข่าย เซิร์ฟเวอร์ แอปพลิเคชัน และอุปกรณ์ปลายทางนั้นมีขนาดมหาศาลและเพิ่มขึ้นในอัตราเร่ง  ปัญหาหลักที่นักวิเคราะห์ความปลอดภัยต้องเผชิญคือ "ภาวะอ่อนล้าจากการแจ้งเตือน" (Alert Fatigue) ซึ่งเกิดจากการที่ระบบ Security Information and Event Management (SIEM) แบบดั้งเดิมมักสร้างการแจ้งเตือนที่เป็น False Positive หรือแจ้งเตือนที่ไม่สำคัญจำนวนมาก  จากผลการศึกษาพบว่านักวิเคราะห์ความปลอดภัยในศูนย์ปฏิบัติการความปลอดภัย (SOC) กว่า 70% ประสบภาวะหมดไฟ (Burnout) และ 65% พิจารณาที่จะลาออกภายในหนึ่งปีเนื่องจากความเครียดในการไล่ตามการแจ้งเตือนที่ไม่มีที่สิ้นสุด  
การตรวจสอบ Log ในปัจจุบันยังคงอาศัยการเขียนกฎเกณฑ์ที่ตายตัว (Rule-based) และการใช้ Regular Expressions (Regex) ในการแยกวิเคราะห์ข้อมูล ซึ่งมีความเปราะบางและไม่สามารถปรับตัวตามรูปแบบการโจมตีใหม่ๆ (Novel Threats) ได้  เมื่อเกิดเหตุการณ์ผิดปกติ นักวิเคราะห์ต้องใช้เวลาเฉลี่ย 30 ถึง 60 นาทีในการรวบรวมบริบท (Context) ของการแจ้งเตือนเพียงหนึ่งรายการ โดยต้องสลับไปมามากกว่า 5 หน้าจอเพื่อตรวจสอบข้อมูลจากแหล่งที่มาที่แตกต่างกัน  ภายในบริบทของประเทศไทย ปัญหานี้ทวีความรุนแรงขึ้นจากการขาดแคลนบุคลากรด้านไซเบอร์กว่า 20,000 - 30,000 ตำแหน่ง ทำให้องค์กรขนาดกลางและขนาดย่อม (SMEs) ไม่สามารถเข้าถึงการเฝ้าระวังความปลอดภัยที่มีคุณภาพได้เนื่องจากต้นทุนที่สูงเกินไป

## Solution  
AI Agent ไม่เพียงแต่ทำหน้าที่เป็นผู้ช่วยในการค้นหาข้อมูล แต่ยังสามารถแสดงบทบาทเป็น "Agentic SOC Analyst" ที่ทำงานได้กึ่งอัตโนมัติ  เมื่อตรวจพบ Log ที่น่าสงสัยผ่านระบบการตรวจสอบแบบเรียลไทม์ AI สามารถเริ่มกระบวนการสืบสวนเบื้องต้นได้ทันที เช่น การตรวจสอบประวัติการเข้าถึงข้อมูลของบัญชีผู้ใช้นั้นๆ การตรวจสอบชื่อเสียงของไอพีปลายทางจากฐานข้อมูลภัยคุกคาม (Threat Intelligence) และการสรุปความเสี่ยงเบื้องต้นให้ Admin ทราบพร้อมแนวทางการแก้ไข (Remediation steps)
- การแจ้งเตือนเหตุละเมิดทันที: ช่วยให้องค์กรสามารถรายงานเหตุต่อ กคส. ได้ภายใน 72 ชั่วโมงตามกฎหมาย    
- การตรวจสอบการเข้าถึงที่ผิดนโยบาย: เช่น พนักงานพยายามดาวน์โหลดข้อมูลลูกค้าจำนวนมากผิดปกติเพื่อไปขายต่อ (Insider Threat)    
- การจัดเก็บประวัติการทำงานเพื่อการ Audit: สร้างเส้นทางหลักฐาน (Audit Trail) ที่พร้อมรับการตรวจสอบจากเจ้าหน้าที่    

## ทำอะไร & เลือกหัวข้อไหน (Project Overview & Track)

ชื่อโปรเจกต์: AI Agent Log Assistant (ผู้ช่วยวิเคราะห์ Log อัจฉริยะ)

หัวข้อการแข่งขัน (Track): 2. Security Operations & Governance

เหตุผลที่เข้าข่าย: เป็นการนำ AI มาช่วยสนับสนุนการปฏิบัติการ (Operations) ลดภาระผู้ดูแลระบบ และช่วยวิเคราะห์เหตุการณ์ภัยคุกคาม (Incident Response) ได้รวดเร็ว

## ทำไมเลือกหัวข้อนี้ & ปัญหาคืออะไร (The Problem)

"Data Rich, but Insight Poor" (มีข้อมูลเยอะ แต่ขาดความเข้าใจ)

Log Overload: ในแต่ละวัน องค์กรขนาดกลางถึงใหญ่มี Log เกิดขึ้นมหาศาล (Millions of lines/day) ทั้งจาก Server, Firewall และ Application

Human Limitation: มนุษย์ไม่สามารถอ่าน Log ทีละบรรทัดได้ทันเวลา และต้องใช้ผู้เชี่ยวชาญ (Specialist) ที่มีทักษะสูงในการเขียน Query เพื่อค้นหาความผิดปกติ

Reactive Response: กว่าจะรู้ตัวว่าถูกโจมตี ข้อมูลก็มักจะรั่วไหลออกไปแล้ว (High MTTR - Mean Time To Respond) หรือเกิดความเสียหายไปแล้ว

## เราแก้ปัญหาอย่างไร (Our Solution)

เราสร้าง AI Agent Log ซึ่งเปรียบเสมือน "AI SOC Analyst เพื่อนคู่ใจ Admin" ที่ทำงานได้ 24 ชั่วโมง

Concept: เปลี่ยนการตรวจสอบ Log ที่ซับซ้อน ให้กลายเป็นการ "พูดคุย" (Conversational Interface)

Core Mechanism: ใช้เทคโนโลยี Generative AI (LLM) ทำงานร่วมกับ RAG (Retrieval-Augmented Generation) เพื่ออ่านและวิเคราะห์ Log ขององค์กร แล้วสรุปผลออกมาเป็นภาษามนุษย์

## สิ่งที่จะทำ & การทำงาน (How It Works & Key Features)

ระบบของเราประกอบด้วย 3 ส่วนหลัก:

- Log Ingestion: รับไฟล์ Log (เช่น Apache Access Log, Auth Log) เข้าสู่ระบบ

- AI Analysis (The Brain):

  - ใช้ LLM (เช่น Gemini 1.5 Pro หรือ GPT-4o) ในการทำความเข้าใจบริบท

  - ใช้เทคนิค RAG ดึงข้อมูลที่เกี่ยวข้องมาวิเคราะห์เพื่อป้องกันการตอบมั่ว (Hallucination)

  - มี Risk Scoring System ประเมินความรุนแรง (Low, Medium, High, Critical)

- Interactive Dashboard:

  - Natural Language Query: Admin พิมพ์ถามได้เลย เช่น "มี IP ไหนพยายาม Brute Force ช่วง 10 นาทีที่ผ่านมาไหม?"

  - Auto-Recommendation: AI แนะนำวิธีแก้ไขทันที เช่น "แนะนำให้ Block IP 1.2.3.4 ผ่าน Firewall ด้วยคำสั่ง..."

## ประโยชน์ & กลุ่มเป้าหมาย (Benefits & Target Users)

### ประโยชน์ที่ได้รับ (Value Proposition):

- Speed: ลดเวลาการตรวจสอบ (Investigation Time) จากชั่วโมงเหลือเพียงวินาที

- Accessibility: Admin ทั่วไป (General IT) สามารถทำหน้าที่ Security Analyst ได้ โดยไม่ต้องเรียนรู้ Query Language ยากๆ

- Proactive: ตรวจเจอความผิดปกติที่ซ่อนอยู่ได้เร็วกว่าการรอให้เกิดความเสียหาย

### กลุ่มเป้าหมาย (Target Audience):

- SMEs & Startups: บริษัทที่มีงบประมาณจำกัด ไม่สามารถจ้างทีม SOC ขนาดใหญ่ได้

- Education & Government: หน่วยงานที่มีข้อมูลสำคัญ (PDPA) แต่ขาดแคลนบุคลากรด้าน Security

- IT Admin / DevOps: ผู้ดูแลระบบที่ต้องการเครื่องมือช่วยเฝ้าระวังความปลอดภัย
