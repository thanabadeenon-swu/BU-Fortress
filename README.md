### Track: Security Operations & Governance (SOG)
# Product: Ai Log Agent
## Problem  
ในโครงสร้างพื้นฐานไอทีสมัยใหม่ ปริมาณของข้อมูล Log ที่เกิดขึ้นจากระบบเครือข่าย เซิร์ฟเวอร์ แอปพลิเคชัน และอุปกรณ์ปลายทางนั้นมีขนาดมหาศาลและเพิ่มขึ้นในอัตราเร่ง  ปัญหาหลักที่นักวิเคราะห์ความปลอดภัยต้องเผชิญคือ "ภาวะอ่อนล้าจากการแจ้งเตือน" (Alert Fatigue) ซึ่งเกิดจากการที่ระบบ Security Information and Event Management (SIEM) แบบดั้งเดิมมักสร้างการแจ้งเตือนที่เป็น False Positive หรือแจ้งเตือนที่ไม่สำคัญจำนวนมาก  จากผลการศึกษาพบว่านักวิเคราะห์ความปลอดภัยในศูนย์ปฏิบัติการความปลอดภัย (SOC) กว่า 70% ประสบภาวะหมดไฟ (Burnout) และ 65% พิจารณาที่จะลาออกภายในหนึ่งปีเนื่องจากความเครียดในการไล่ตามการแจ้งเตือนที่ไม่มีที่สิ้นสุด  
การตรวจสอบ Log ในปัจจุบันยังคงอาศัยการเขียนกฎเกณฑ์ที่ตายตัว (Rule-based) และการใช้ Regular Expressions (Regex) ในการแยกวิเคราะห์ข้อมูล ซึ่งมีความเปราะบางและไม่สามารถปรับตัวตามรูปแบบการโจมตีใหม่ๆ (Novel Threats) ได้  เมื่อเกิดเหตุการณ์ผิดปกติ นักวิเคราะห์ต้องใช้เวลาเฉลี่ย 30 ถึง 60 นาทีในการรวบรวมบริบท (Context) ของการแจ้งเตือนเพียงหนึ่งรายการ โดยต้องสลับไปมามากกว่า 5 หน้าจอเพื่อตรวจสอบข้อมูลจากแหล่งที่มาที่แตกต่างกัน  ภายในบริบทของประเทศไทย ปัญหานี้ทวีความรุนแรงขึ้นจากการขาดแคลนบุคลากรด้านไซเบอร์กว่า 20,000 - 30,000 ตำแหน่ง ทำให้องค์กรขนาดกลางและขนาดย่อม (SMEs) ไม่สามารถเข้าถึงการเฝ้าระวังความปลอดภัยที่มีคุณภาพได้เนื่องจากต้นทุนที่สูงเกินไป

## Solution  
AI Agent ไม่เพียงแต่ทำหน้าที่เป็นผู้ช่วยในการค้นหาข้อมูล แต่ยังสามารถแสดงบทบาทเป็น "Agentic SOC Analyst" ที่ทำงานได้กึ่งอัตโนมัติ  เมื่อตรวจพบ Log ที่น่าสงสัยผ่านระบบการตรวจสอบแบบเรียลไทม์ AI สามารถเริ่มกระบวนการสืบสวนเบื้องต้นได้ทันที เช่น การตรวจสอบประวัติการเข้าถึงข้อมูลของบัญชีผู้ใช้นั้นๆ การตรวจสอบชื่อเสียงของไอพีปลายทางจากฐานข้อมูลภัยคุกคาม (Threat Intelligence) และการสรุปความเสี่ยงเบื้องต้นให้ Admin ทราบพร้อมแนวทางการแก้ไข (Remediation steps)
- การแจ้งเตือนเหตุละเมิดทันที: ช่วยให้องค์กรสามารถรายงานเหตุต่อ กคส. ได้ภายใน 72 ชั่วโมงตามกฎหมาย    
- การตรวจสอบการเข้าถึงที่ผิดนโยบาย: เช่น พนักงานพยายามดาวน์โหลดข้อมูลลูกค้าจำนวนมากผิดปกติเพื่อไปขายต่อ (Insider Threat)    
- การจัดเก็บประวัติการทำงานเพื่อการ Audit: สร้างเส้นทางหลักฐาน (Audit Trail) ที่พร้อมรับการตรวจสอบจากเจ้าหน้าที่
